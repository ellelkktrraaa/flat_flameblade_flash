# 焰刃流光 · 如何做一个新角色

这是一份写给**没有编程经验**的朋友看的指南。你不需要看懂代码——只要跟着步骤做，就能给游戏加一个全新的角色。

---

## 先理解：角色是怎么"装进"游戏的？

想象一下：这个游戏就像一个**商场**，每个角色文件就是一个**店铺**。游戏的系统（`js/systems/` 和 `js/` 开头的那些文件）是商场的**基础设施**——走廊、灯光、收银台。店铺只需要开到商场里，基础设施就会自动服务它。

### 三样东西决定一个角色

要把一个角色"装进"游戏，只需要三样东西：

```
① 一个文件     ←  角色的"身份证"，放在 js/characters/ 里
② 一份配置     ←  在上述的那个文件里填几个配置，告诉游戏"这个角色什么样"
③ 一行引用     ←  在 index.html 里写一行，告诉游戏"登录这个人的身份信息"
```

### 角色文件和游戏其他部分的关系

```
index.html            ← 入口：你在这里"注册"角色（加一行 <script>）
    │
    ├─ 加载 js/characters/xxx.js   ← 角色文件（你需要写的唯一文件）
    │       │
    │       ├─ CHAR_CONFIGS     = 角色的"简历"（名字、血量、速度、贴图）
    │       ├─ CHAR_SKILL_FACTORIES = 角色的"技能表"（按什么键放什么技能）
    │       └─ hooks（可选）    = 角色的"特殊能力"（暴击、闪避、飞行…）
    │
    └─ 加载 js/ 和 js/systems/   ← 系统文件（你不需要碰的东西）
            │
            ├─ 它们会自动读取 CHAR_CONFIGS，把角色显示在选人界面
            ├─ 它们会在战斗时自动调用技能
            ├─ 它们会在每一帧检查 hooks，执行特殊能力
            └─ 它们负责网络联机、AI、画面渲染等一切杂事
```

**关键点**：角色文件和系统文件是**完全解耦**的。什么意思呢？

- 系统文件**不认识**任何具体角色——它们只读一个叫 `CHAR_CONFIGS` 的登记表
- 你在角色文件里往登记表填了信息，系统自动就知道了
- 就像你去银行开户——银行系统不关心你是谁，你填了表格，它自动给你办业务

### 什么是"钩子"（hooks）？

钩子是角色文件里最灵活的部分。它就像店铺里的**定制服务窗口**——你不开这个窗口，店铺走默认服务；你开了，就可以做额外的事情。

举个例子：刺客的"一瞬"技能可以无敌闪避——

- 刺客的店铺里开了一个 `onDashContact` 钩子窗口
- 游戏系统每帧都在问："有店铺开了突进碰撞窗口吗？"
- 刺客回答："有！我碰人不用默认伤害"
- 其他角色没开这个窗口，系统就走默认流程

这不需要你理解代码。你只需要知道：**想要什么特殊能力，告诉会编程的朋友开哪个钩子就行**。后文会列出所有可用的钩子。

---

## 你需要准备什么

- 一个**文本编辑器**
- **角色的贴图**：就是角色的"照片"，至少需要 4 张──站着、走路、跳起来、攻击（你可以找会画画的朋友帮忙，或者用现成的素材）
- **一个英文代号**：比如你做的角色是"龙骑士"，那代号就叫 `dragonknight`（不用大写、不用空格）

---

## 第一步：看看别人是怎么做的

打开游戏文件夹，找到 `js/characters/` 这个目录（目录就是文件夹的意思）。里面有 6 个文件，每个文件就是一个角色：

```
js/characters/
├── knight.js      ← 骑士（最简参考，没有特殊机制）
├── mage.js        ← 法师
├── archer.js      ← 弓箭手（有特殊属性：箭矢数量）
├── paladin.js     ← 圣骑士（有特殊机制：受伤回能量、空中用技能）
├── witch.js       ← 魔女（有特殊机制：陨石爆炸、飞行）
└── assassin.js    ← 刺客（最复杂参考，有 8 种特殊机制）
```

**建议用记事本打开 `knight.js` 看看**——它是最简单的，只有基本属性 + 3 个技能，没有任何花活。

---

## 第二步：复制一个模板

最简单的方法：复制 `knight.js`，重命名为你的角色代号。比如 `dragonknight.js`。

用文本编辑器打开它，你会看到一大堆花括号和符号。别怕，我们只需要改几个地方。

---

## 第三步：改属性（告诉游戏"这个角色是什么样的"）

找到文件开头这一段：

```js
CHAR_CONFIGS.knight = {
    id: 'knight',
    name: '骑士',
    hp: 100,
    ...
}
```

把它改成你的角色信息：

| 改哪里 | 说明 | 例子 |
|--------|------|------|
| `.knight` 的 `knight` | 改成你的英文代号 | `.dragonknight` |
| `id: 'knight'` | 改成你的代号 | `id: 'dragonknight'` |
| `name: '骑士'` | 改成角色的中文名 | `name: '龙骑士'` |
| `hp: 100` | 血量（数字越大越肉） | `hp: 120` |
| `maxEnergy: 100` | 能量上限 | 不变也行 |
| `resourceLabel: '圣光'` | 能量条的名字（默认是"能量"，圣骑士改成"圣光"） | 按需修改 |
| `speed: 2.25` | 移动速度（越大跑越快） | `speed: 2.0` |
| `attackDamage: 5` | 普攻伤害 | `attackDamage: 6` |

**重点：改贴图路径**

你会看到这一段：

```js
images: {
    idle: IMG.knight_idle,
    walk: IMG.knight_walk,
    jump: IMG.knight_jump,
    attack: IMG.knight_attack,
    ult: IMG.knight_ult,
},
```

如果你有新的贴图，需要先把图片放到 `assets/` 文件夹里，然后修改这里的引用。**如果你暂时没有新贴图，不要改这里**——角色会先用骑士的外观，不影响测试。

---

## 第四步：改技能（告诉游戏"这个角色能干什么"）

找到 `CHAR_SKILL_FACTORIES` 这一段。每个角色有 4 个技能：

| 键位 | 在代码里叫什么 | 说明 |
|------|---------------|------|
| J 键 / 攻击按钮 | `'attack'` | 普通攻击 |
| U 键 / 技能1按钮 | `'skill1'` | 第一个技能 |
| I 键 / 技能2按钮 | `'skill2'` | 第二个技能 |
| O 键 / 大招按钮 | `'ult'` | 大招 |

每个技能都有这几项：

```js
new Skill({
    key: 'skill1',        // 技能名（skill1 / skill2 / ult）
    name: '剑气',          // 显示在技能面板的名字
    cooldown: 480,         // 冷却时间（数字越大等越久）
    energyCost: 20,        // 消耗多少能量
    canUse: (owner) => {   // 什么时候可以用
        return true;
    },
    execute: (owner) => {  // 用了之后发生什么
        // 这里是具体效果...
        return { success: true };
    }
}),
```

**最简单的改法**：只改 `name` 和 `cooldown`，不改效果。名字改了在游戏里就能看到，冷却时间改了能感受到区别。

---

## 第五步（补充）：填写图鉴信息

角色选人界面会自动展示你的角色，但游戏里还有一个"图鉴"功能——点右上角的📖按钮可以看到。图鉴中的角色介绍、技能描述也是写在角色文件里的，不需要改系统文件。

在 `CHAR_CONFIGS.xxx` 的最后加一个 `dex` 字段：

```js
dex: {
    icon: '🐉',          // 图鉴里显示的图标（可以到处找emoji）
    intro: '龙骑士的简短介绍……\n"这是角色台词"',
    stats: [
        { label: '生命', value: '120' },
        { label: '能量上限', value: '100' },
    ],
    skills: [
        { name: '技能名', desc: '技能描述', meta: '消耗：20 ｜ 冷却：8秒' },
        { name: '技能名2', desc: '描述2', meta: '消耗：无 ｜ 冷却：1秒' },
        // ... 写 3~5 个技能
    ],
},
```

**最简做法**：随便写个图标、一段介绍、几个技能名字就行，不需要很精确。以后再补也可以的喵～

---

## 第五步：注册角色

打开游戏文件夹里的 `index.html`（用记事本打开），找到一大堆 `<script>` 标签的地方。在角色那一段的最后加一行：

```html
<script src="js/characters/knight.js"></script>
<script src="js/characters/mage.js"></script>
...
<script src="js/characters/dragonknight.js"></script>   <!-- ← 你新加的 -->
```

**就这一行，不需要改任何别的文件。**

---

## 第六步：试玩！

用浏览器打开 `index.html`，选择难度，你应该能在角色选择界面看到你的新角色了。选中它，点击"开始战斗"，看看能不能正常跑起来。

---

## 进阶：给角色加"特殊能力"

如果上面的基础版本跑通了，你可能想给角色加一些特殊效果。每个角色文件末尾可以有一个叫 `hooks`（钩子）的东西——就像乐高积木的接口，你可以把自定义的逻辑"挂"上去。

你只需要告诉程序员朋友你想要的**时机**和**效果**，他们会帮你填代码。

这里列出所有可用的"时机"：

### 每帧会触发的

| 时机 | 什么意思 |
|------|----------|
| `onUpdate` | 游戏的每一帧都会跑一次。用于需要持续监测的东西 |
| `onFighterUpdate` | 角色的物理引擎每帧跑完时触发。用于计时器、buff倒计时 |

### 画面绘制时触发的

| 时机 | 什么意思 |
|------|----------|
| `onFighterDraw` | 角色贴图画完之后，可以画额外的特效（刀光、残影、蓄力条） |
| `onOverlayDraw` | 所有画面元素画完之后，可以画全屏覆盖效果（大招黑幕、滤镜） |
| `onHUD` | 更新血条/能量条界面时触发。可以改能量条颜色、加额外的UI |

### 战斗交互时触发的

| 时机 | 什么意思 |
|------|----------|
| `onProjectileUpdate` | 你发射的飞行物每帧更新时触发。可以控制飞行物特殊行为 |
| `onProjectileHit` | 你的飞行物撞到敌人时触发。可以穿透、爆炸、生成火焰等 |
| `onDashContact` | 冲刺撞到敌人时触发。可以决定冲刺是否有伤害 |
| `onDamageDealt` | 你对敌人造成伤害时触发。可以加暴击、额外伤害 |
| `onDamageReceived` | 你受到伤害时触发。可以回血、回能量、减伤 |

### 开局/拾取时触发的

| 时机 | 什么意思 |
|------|----------|
| `onPickup` | 你捡到地上的道具时触发。可以额外回箭矢、回特殊资源 |
| `initFighter` | 每局开始时触发。可以初始化角色的特殊状态 |
| `initResources` | 游戏加载时触发一次。可以预加载角色的特殊图片 |

### 举个例子

如果你想做一个"忍者"，被攻击时有概率闪避——就告诉程序员朋友：**给 `onDamageReceived` 加一个闪避概率**。

如果你想做一个"召唤师"，普攻会召唤一个跟随的小精灵——就告诉程序员朋友：**给 `onUpdate` 加一个小精灵跟随逻辑，给 `onFighterDraw` 加小精灵的绘制**。

---

## 总结：加一个新角色要做什么

```
1. 复制 knight.js → 重命名为 dragonknight.js
2. 改 CHAR_CONFIGS 里的属性（名字、血量、贴图路径）
3. 改 CHAR_SKILL_FACTORIES 里的技能名和冷却
4. 在 index.html 里加一行 <script src="...dragonknight.js">
5. 用浏览器打开 index.html 测试
6. （可选）告诉程序员朋友你想加的特殊能力
```

**不改任何 js/systems/ 和 js/ 开头的系统文件**——这就是插件架构的意义。
